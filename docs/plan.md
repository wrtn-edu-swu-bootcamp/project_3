# 배달앱 리뷰 주간 분석 & 액션 제안 서비스 MVP 기획안

## 요약

소상공인/사장님은 주문 폭주로 바쁘고 디지털 약자가 많아 리뷰를 제대로 확인하고 분석하기 어렵다. 40~70대 배달앱 운영 자영업자를 타겟으로, 매주 1회 리뷰를 자동 수집·분류하고 긴급/고려/잘함 3단계 우선순위 액션을 제안한다. 월간 리포트는 늦지만 주간 대응은 손님 이탈 전 개선 기회를 제공한다. MVP는 독립형 모바일 웹이며, 배달의민족 크롤링 기반 무료 서비스로 시작해 향후 B2B 구독(월 1~2만원)으로 확장한다.

---

## 1. 서비스 한눈에 보기

### 문제 정의
소상공인/사장님들은 ①주문 폭주로 시간이 없고 ②디지털 약자(40~70대 다수)여서 앱 사용이 어려우며 ③리뷰를 읽어도 무엇을 고쳐야 할지 판단하기 어렵다. 리뷰 방치 시 부정 평가 누적으로 매출이 하락하지만, 대응이 늦어 손님이 떠난 후에야 알게 된다.

### 타겟
배달앱 운영 소상공인/사장님(특히 40~70대). 바쁜 자영업자, 디지털 약자, 리뷰 분석 능력 부족 유형 포함.

### 한 줄 가치 제안
**"손님 떠나기 전, 1주마다 고칠 점을 알려드립니다."**

---

## 2. MVP가 제공하는 결과물(주간 리포트)

**리포트 구성**
- 수집 기간(최근 7일)
- 🚨 긴급 개선 필요(1~2개, 있을 때만)
- ⚠️ 개선 고려(0~3개)
- ✅ 잘하고 있음(0~3개)
- 카테고리별 상세(음식 품질/배달·서비스/포장/가격/기타): 긍·부정 비율(%), 리뷰 수(n), 키워드 Top 3, 부정 리뷰 예시

**설계 원칙**: 3분 내 파악 가능 / "긴급 우선" 배치 / 모바일 웹 최적화

---

## 3. 핵심 기능(MVP)

### 3.1 가게 URL 등록 + 주간 리뷰 수집

**초기 세팅**: 배민 가게 URL 입력(1회)

**주간 수집 규칙**(최근 7일 기준)
- **최소 10개**: 미만 시 "표본 부족(참고용)" 배지 표시
- **권장 50개**: 주간 패턴 판단과 5개 카테고리 분류가 의미 있는 기준
- **최대 100개**: 리뷰가 많은 가게에서 50개만 보면 이벤트/쏠림 편향 가능

**확장 수집 조건**(50 → 100개)
- 쿠폰/이벤트 언급 급증
- 짧은 칭찬 반복("맛있어요"만 다수) 패턴 과도
- 특정 날짜 리뷰 몰림 등 "쏠림" 감지

**수집 데이터**: 텍스트, 별점, 작성 날짜  
**생성 시간**: 보통 30~60초, 최대 90초

### 3.2 리뷰 분류 + 요약

**5개 카테고리 자동 분류**: 음식 품질 / 배달·서비스 / 포장 / 가격 / 기타

**카테고리별 출력**: 긍·부정 비율(%), 리뷰 수(n), 키워드 Top 3, 부정 리뷰 예시

### 3.3 우선순위 액션 제안(Killer Feature)

**3단계 우선순위 자동 판정(비율 중심)**

- 🚨 **긴급 개선 필요**(1~2개): 카테고리 부정 비율 ≥ 30%
- ⚠️ **개선 고려**(0~3개): 부정 비율 15~29%
- ✅ **잘하고 있음**(0~3개): 긍정 비율 ≥ 70%
- 단, 해당 카테고리 언급 < 10개이면 "확신 낮음" 배지 우선 표시 + 액션 개수 축소

**액션 제안 방식**
- 카테고리별 **액션 템플릿(액션 뱅크)**에서 선택
- 각 이슈마다 **추천 1개 + 대안 2개**(총 3개) 제안
- 긴급(🚨)은 추천 1개 강조, 대안은 접어두기(선택적 확인)

**신뢰도 배지**
- "확신 낮음": 해당 카테고리 언급 < 10개
- "표본 부족(참고용)": 총 리뷰 < 10개
- "이벤트 가능성(참고)": 쿠폰/이벤트/짧은 칭찬 반복 등 감지 시 안내

## 4. User Journey

**대표 사용자**: 58세 치킨집 사장님, 하루 주문 많아 바쁨, 스마트폰 서툼

**주간 사이클**

1. **서비스 접속**: 배민 리뷰 읽기 힘듦
2. **가게 URL 등록**(1회): 배민 URL 복사·붙여넣기
3. **리포트 생성 클릭**(매주): 30~60초 대기
4. **🚨 긴급 확인**: 있으면 즉시 파악 → 추천 액션 확인, 필요시 대안 2개 펼쳐보기
5. **⚠️ 고려/✅ 잘함 확인**: 이번 주 할 일 정리, 잘하는 점 확인
6. **다음 주 재확인**: 개선 효과 체감

## 5. 기존 기능 vs 우리 서비스

| 구분 | 배달앱 기본 기능 | 우리 서비스(MVP) |
|---|---|---|
| 리뷰 확인 | 목록을 직접 읽음 | 주간 요약 리포트 |
| 핵심 이슈 파악 | 사장님이 직접 판단 | 3단계 우선순위 자동 판정 |
| 실행 가이드 | 없음 | 추천 1개 + 대안 2개 제안 |
| 신뢰도 신호 | 없음 | 표본 부족/확신 낮음/이벤트 가능성 안내 |

**차별화 포인트**: 주간 주기(빠른 대응) / 액션 중심(할 일 전환) / 디지털 약자 친화(3분 리포트) / 무료(MVP)

## 6. MVP 성공 지표(KPI)

**2일 MVP 데모/내부 설문 기반**

1. **리포트 생성 성공률**: ≥ 90%
2. **평균 생성 시간**: ≤ 90초
3. **리포트 완독률**: ≥ 70%
4. **"액션 제안 유용" 설문**: 평균 4.0+ (5점) 또는 Yes ≥ 75%

*(MAU, 재방문율은 MVP 이후 추적)*

## 7. MVP 범위 및 제약사항

### 포함
- ✅ 가게 URL 1회 등록
- ✅ 수동 버튼으로 최근 7일 리뷰 수집(최소 10 / 권장 50 / 최대 100, 확장 조건 적용)
- ✅ 5개 카테고리 분류 + 비율 기반 우선순위 판정
- ✅ 3단계 액션 제안(추천 1개 + 대안 2개, 액션 뱅크 기반)
- ✅ 신뢰도 배지(확신 낮음/표본 부족/이벤트 가능성)

### 제외(향후 확장)
- ❌ 로그인/회원가입
- ❌ 자동 스케줄링/알림(카톡/문자)
- ❌ 다중 가게 관리
- ❌ 타 플랫폼(쿠팡이츠/요기요)

### 제약사항
- 생성 시간 30~90초(프로그레스 바)
- 배민 HTML 구조 변경 시 동작 불가 가능성
- 데이터 보관: 최근 4주치만
- 보안: URL 접근(초기 MVP, 향후 로그인 추가)

---

## 8. 리스크 & 대응

### 1. 크롤링 차단
**리스크**: IP 차단/CAPTCHA 발동  
**대응**: 요청 간격 조절, User-Agent 설정, 공식 API 연동 협의

### 2. 인지도 부족
**리스크**: 사용자 유입 없음  
**대응**: 소상공인 커뮤니티 홍보, 배민 사장님 광장 활용

### 3. 보안 이슈
**리스크**: URL 접근으로 개인정보 우려  
**대응**: 공개 정보(리뷰)만 수집 명시, 향후 인증 추가

### 4. 액션 제안 오류
**리스크**: 분석 오류로 신뢰 상실  
**대응**: 피드백 버튼 제공, 설문 기반 개선, 확신 낮음 배지로 투명화

## 9. 향후 확장 가능성

- 자동 스케줄링(매주 월요일 자동 생성)
- 자동 알림(카톡/문자)
- 주간 비교(지난주 대비 개선 추적)
- 다중 가게 관리
- 타 플랫폼 확장(쿠팡이츠/요기요)
- B2B 구독 모델(월 1~2만원)
