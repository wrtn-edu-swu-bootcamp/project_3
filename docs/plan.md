# 배달앱 리뷰 주간 분석 & 액션 제안 서비스 기획안 (MVP)

## 요약

배달앱 운영 소상공인(40~70대)은 바쁜 업무로 리뷰를 제대로 확인하지 못하고, 읽어도 무엇을 어떻게 고쳐야 할지 판단하기 어렵습니다. 본 서비스는 **매주 1회 지난 주(월~일) 리뷰를 정리**하고 **긴급/고려/잘함 3단계 우선순위 액션**을 제안합니다.

**MVP 범위**: 실제 가게 3-5곳의 주간 리포트 데모를 통해 서비스 가치를 증명합니다.

**핵심 가치**: "손님 떠나기 전, 매주 고칠 점을 알려드립니다."

---

## 1. 문제 정의 및 타겟

### 소상공인이 겪는 문제

1. **시간 부족**: 주문 폭주로 리뷰 확인할 시간 없음
2. **디지털 약자**: 앱 사용 서툴고 데이터 해석 어려움
3. **판단 어려움**: 무엇을, 어떻게 고쳐야 할지 모름

**결과**: 리뷰 방치 → 부정 평가 누적 → 손님이 떠난 후에야 알게 됨

### 타겟 사용자

**1차 타겟**: 배달앱 운영 소상공인/사장님 (특히 40~70대)

**대표 페르소나**: 50-60대 치킨집 사장님. 바쁜 주방 운영으로 리뷰 확인 시간 부족. 스마트폰 사용은 전화·카톡 위주. "뭐가 급한 건지, 어떻게 고쳐야 할지 막막해요"

### 시장 기회

- 배달앱 리뷰 관리 전문 서비스는 국내에서 희소함 (주간 분석 + 액션 제안의 차별화)
- 배민/요기요는 리뷰 목록만 제공, 분석·가이드 부재
- 참고 자료에 따르면 리뷰 응답률이 높은 업체는 더 많은 매출 창출, 소비자 대다수가 리뷰 응답 업체를 더 신뢰 (참고 수준: 중간~높음)*

## 2. 서비스 차별점

| 대체재 | 현재 방식 | 한계 | 우리 서비스의 차별점 |
|---|---|---|---|
| **수기 확인** | 리뷰 목록 직접 읽기 | 시간 많이 걸림, 놓치는 이슈 많음 | 3분 요약, 우선순위 안내 |
| **엑셀 정리** | 직원이나 사장님이 수작업 | 분석 능력 필요, 지속 어려움 | 카테고리별 정리, 주간 리포트 제공 |
| **직원에게 맡김** | 아르바이트생에게 리뷰 체크 요청 | 판단력 부족, 일관성 없음 | 객관적 기준 적용, 액션 제안 |
| **리뷰 대행 업체** | 리뷰 응답 대행 서비스 | 비용 높음(월 10만원대), 분석 부재 | 분석 + 액션 제안 포함, 저렴 |
| **배달앱 기본 기능** | 배민/요기요 리뷰 목록 | 필터링만, 분석·가이드 부재 | 카테고리별 정리, 우선순위, 액션 제안 |

**차별화 핵심**:
- **주간 대응**: 손님 이탈 전 개선 골든타임 (1주 = 월간보다 빠르고, 일간보다 실용적)
- **액션 전환**: "문제" → "할 일"로 전환
- **3분 요약**: 긴급 + 이번 주 할 일만 파악
- **투명한 배지**: 판단 근거의 한계를 안내 (표본 부족/확신 낮음/이벤트 가능성)

## 3. MVP 정의 및 주간 리포트 구성

### MVP 범위

**제공**: 샘플 가게 3-5곳(A치킨, B치킨 등 비식별)의 주간 리포트 데모

**미제공**: 가게 등록, 정기 리포트, 알림, 로그인

**목적**: 완성된 샘플을 통해 서비스 가치 증명

### 주간 리포트 구조

**설계 원칙**: 3분 내 파악 가능 / 긴급 우선 배치 / 모바일 최적화

**리포트 기간**: 주차 단위(월~일), 지난 주 리뷰 기준 (예: "1월 12일(월)~1월 18일(일) 주간 리포트")

#### 상단: 한눈에 보기

```
[가게명] 지난 주 리포트
1월 12일(월) ~ 1월 18일(일) | 리뷰 67개

📌 한 줄 결론: "배달 시간 지연 문제가 심각합니다."

🚨 긴급 개선 필요
[배달·서비스] 배달 시간 지연 (부정 60%)
💡 추천: 피크타임 배달 인력 추가 배치
📂 대안 2개 보기 (접기/펼치기)
```

- 긴급은 빨간색 강조, 카드 크기 2배
- 추천 액션은 즉시 표시, 대안은 클릭 시 확인
- 긴급이 없으면 "🎉 이번 주는 긴급한 문제가 없습니다!"

#### 중단: 카테고리별 상세

- **5개 카테고리**: 음식 품질 / 배달·서비스 / 포장 / 가격 / 기타
- **카테고리별 제공**: 긍정/부정 비율, 리뷰 수, 대표 표현, 부정 리뷰 예시 (최대 3개)
- **참고 정보**: 배달 경험(알뜰배달, 한집배달 등) 관련 언급이 많은 경우 참고로 표시

#### 하단: 배지 및 안내

- **신뢰도 배지**:
  - 표본 부족(참고용): 총 리뷰 < 10개
  - 확신 낮음: 특정 카테고리 < 10개
  - 이벤트 가능성(참고): 특정 시기 리뷰 몰림
- **안내**: "리뷰가 서비스에 반영되는 시점에 따라 일부 리뷰가 누락될 수 있습니다"
- **면책**: "최종 결정은 사장님께 있습니다. 가게 상황을 고려하여 조정해주세요."

## 4. 핵심 기능

### 4.1 주간 리뷰 정리

- 지난 주(월~일) 리뷰를 5개 카테고리로 정리
- 카테고리별 긍정/부정 경향 표시
- 부정 리뷰 예시 제공

**리뷰 데이터 기준**:
- 최소 10개: 미만 시 "표본 부족" 배지
- 권장 50개: 주간 패턴 판단에 적합

### 4.2 우선순위 액션 제안

**3단계 우선순위**:
- 🚨 **긴급 개선 필요** (부정 비율 ≥ 30%, 최대 1-2개)
- ⚠️ **개선 고려** (부정 비율 15-29%, 최대 0-3개)
- ✅ **잘하고 있음** (긍정 비율 ≥ 70%, 최대 0-3개)

**제공 방식**: 각 이슈마다 추천 1개 + 대안 2개

### 4.3 액션 뱅크 (예시)

카테고리별 이슈 유형과 대응 방법을 정리한 지침서:
- 배달·서비스: 배달 지연 → 피크타임 인력 추가 / 대행 업체 등록 / 주문 중단 설정
- 음식 품질: 온도 낮음 → 보온 포장재 교체 / 온도 재확인 절차 / 즉시 포장 원칙
- 포장: 포장 파손 → 이중 포장 / 완충재 추가 / 용기 업그레이드
- 가격: 가격 불만 → 할인 쿠폰 / 세트 가격 조정 / 리뷰 작성 사은품

각 액션은 예상 비용(낮/중/높), 소요 시간(짧/중/김), 난이도(하/중/상), 기대 효과, 주의사항 포함

## 5. 사용자 여정 (MVP)

**대표 사용자**: 50-60대 치킨집 사장님

**시나리오**:
1. 데모 페이지 접속 → 샘플 리포트 선택 (A치킨, B치킨 등)
2. 긴급 카드 확인 → "배달 시간 지연" 빨간색 강조
3. 추천 액션 확인 → "피크타임 배달 인력 추가 배치"
4. 대안 확인 (2개) → 실행 가능성 판단
5. 가치 확인 → "우리 가게도 이렇게 받으면 좋겠다"

---

## 6. 운영 전제 및 검증

### 데이터 확보 (MVP)

- 운영자가 외부 수집 도구(HashScraper)를 통해 샘플 가게 3-5곳의 리뷰를 확보
- 확보된 리뷰 데이터를 엑셀 파일로 정리하여 리포트 생성·제공
- 비식별 처리: A치킨/B치킨 등으로 표시
- 수집 범위: 공개된 리뷰 텍스트, 별점, 작성 시점 (개인정보 제외)

**운영 원칙**: 리포트 기간은 리뷰 작성 시점 기준으로 판단

**리스크**: 외부 도구 정책 변경 시 데이터 제공 중단 가능성 존재

### MVP 검증

**검증 방법**: 샘플 리포트 평가 (내부 팀원 + 자영업자 인터뷰)

**핵심 지표**:
- 이해도: 3분 내 "이번 주 문제/할 일"을 말할 수 있는가? (목표 ≥ 80%)
- 유용성: 제안 액션이 실행 가능해 보이는가? (목표 ≥ 75%)
- 신뢰: 배지/근거가 납득되는가? (목표 ≥ 75%)
- 재사용 의향: 내 가게에도 쓰고 싶은가? (목표 ≥ 70%)
- 가격 수용성: 월 1~2만원 결제 의향? (목표 ≥ 60%)

## 7. 향후 확장 (Phase 2, 참고)

**서비스화 방향**: 사용자가 자신의 가게를 등록하면 매주 월요일(주초) 지난 주 리포트를 정기적으로 제공

**추가 기능**: 가게 등록, 매주 자동 리포트, 알림(카톡/문자), 주간 비교, 리뷰 응답 템플릿

**수익 모델**: 유료 구독 (베이직 월 9,900원 / 스탠다드 월 19,900원 / 프리미엄 월 39,900원)

**장기 확장**: 다중 가게, 타 플랫폼(요기요/쿠팡이츠), 경쟁사 벤치마킹, 소상공인 종합 경영 지원 플랫폼
