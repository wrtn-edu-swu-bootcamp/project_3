# 리액션(RE:ACTION) Wireframes

> 기획안: [plan.md](plan.md)

---

## 설계 원칙

**브랜드**: 리액션 (RE:ACTION) - 사장님을 위한 리뷰 비서

**핵심 가치**: 이번 주 먼저 고칠 일

**UI 톤**: 모던하고 깔끔 / 둥근 모서리 / 넉넉한 여백 / 짧고 친절한 텍스트

**타겟 배려**:
- 40-70대 소상공인 사장님
- 3분 내 파악 가능
- 우선순위 중심 (🚨 긴급 → ⚠️ 고려 → ✅ 잘함)

**접근성 원칙**:
- 큰 글자 (18px+), 큰 터치 영역 (44px+)
- 색상+아이콘+텍스트 라벨 병행
- 펼침/접힘 명확 표시 ("보기 ▾" / "접기 ▴")
- 짧은 문장 (20자 내외)

### 주요 컴포넌트

**히어로 영역** (3줄 룰):
- 서비스명: 리액션
- 1줄: 사장님을 위한 리뷰 비서
- 2줄: 이번 주 먼저 고칠 일
- 3줄 (보조): 3분 요약으로 보여드려요
- 서브: RE:ACTION (작게)

**배지/안내**:
- [⏱️ 3분 요약] 배지
- 안심 안내: "로그인 없이 체험"

**샘플 가게 카드** (3-5개):
- 가게명 + 아이콘
- 업종
- 메타라인: 기간 • 리뷰 수 (통일된 구분점)
- 우선순위 칩 (분리): [🚨긴급 1] [⚠️고려 2] [✅잘함 3]
- CTA: "샘플 리포트 보기"

### 핵심 인터랙션

- 카드 클릭 → Screen 2로 이동
- 우선순위 칩은 정보 표시용 (클릭 불가)

### 접근성

- 큰 글자 (18px+), 둥근 모서리 카드, 터치 영역 44px+, 아이콘+텍스트 라벨로 명확한 구분

### 주요 컴포넌트

**헤더**: ← 뒤로 / 리액션 로고 / 우측 비움

**상단 정보** (2줄): 가게명 + "주간 리포트" / 기간 • 리뷰 수

**한 줄 요약**: 2줄 내로 간결

**먼저 고칠 일**:
- 섹션 제목 + 상태성 보조 문장
  - 긴급 있음: "가장 급한 1가지만 먼저 볼게요"
  - 긴급 없음: "급한 건 없어요. 개선 후보를 볼게요"
- 액션 카드 3단 구조:
  - 상단: 상태 라벨 + 카테고리 (🚨 긴급 · [배달·서비스])
  - 중단: 이슈명 + 수치 (부정 60% • 40건)
  - 하단: 추천 액션 + 칩 2개(비용/시간) + 버튼
- 대안 펼침 시: 난이도 칩 추가

**상세 분석**: 섹션 라벨 + "카테고리 상세 보기 ▾" 버튼 / 5개 카테고리

**참고하세요**:
- 배지 1줄: [확신 낮음] [표본 부족]
- 안내 1줄: "일부 리뷰 누락 가능"
- 면책 접힘: [ ⚖️ 면책 보기 ▾ ] / 펼치면 상세 문구

**피드백 CTA**: "도움이 됐나요?" / [👍 도움돼요] [👎 아쉬워요] / [💬 의견 남기기]

### 핵심 인터랙션

- "대안 보기 ▾" → 펼침 (난이도 칩 추가), "대안 접기 ▴"로 변경
- "카테고리 상세 보기 ▾" → 5개 카테고리 펼침
- "⚖️ 면책 보기 ▾" → 펼침, "면책 접기 ▴"로 변경
- [👍 도움돼요] [👎 아쉬워요] → 반응 저장
- [💬 의견 남기기] → Screen 3

### 접근성

- 터치 영역 44px+, 아이콘+텍스트 라벨 병행, "보기 ▾" / "접기 ▴" 명확 표시, 큰 글자 18px+
